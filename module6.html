// 06.01

The Output Window — Your First Stop

The Output Window (View → Output Window or Ctrl+7) is NinjaTrader's diagnostic console. Every compile error, runtime exception, strategy state change, and Print() message appears here.

What to Look for in the Output Window

  • Red text: Compile errors or runtime exceptions — these require immediate attention
  • Yellow text: Warnings — strategy still runs but may have logic issues
  • White text: Info messages and Print() output
  • Strategy state changes: "Strategy enabled", "Strategy disabled" — confirms strategy lifecycle
✅

Keep the Output Window open and docked at the bottom of your NinjaTrader workspace whenever testing strategies. Critical errors appear here before they manifest as unexpected trading behavior.

Clearing Output Window

Right-click the Output Window → Clear. Do this before each test session to avoid confusion between old and new messages. NinjaTrader log files persist in Documents\NinjaTrader 8\log\ if you need historical records.

// 06.02

Print() for Diagnostic Tracing

The Print() method writes to the Output Window. It's the primary tool for tracing strategy logic, confirming signal conditions, and verifying variable values during execution.

// Basic Print usage
Print("OnBarUpdate fired — Bar: " + CurrentBar);
Print("ATR value: " + atr[0].ToString("F4"));
Print("Position: " + Position.MarketPosition.ToString());

// Conditional Print — only print when signal fires
if (bullCross)
{
    Print(Time[0] + " BULL CROSS detected | Fast: " + fastEma[0].ToString("F2") +
          " | Slow: " + slowEma[0].ToString("F2"));
}

// Print on order fill — use OnExecutionUpdate
protected override void OnExecutionUpdate(Execution execution, ...)
{
    Print("FILL: " + execution.Name + " @ " + execution.Price +
          " Qty: " + execution.Quantity);
}
⚠️

Remove excessive Print() calls before production. Printing on every bar in OnBarUpdate generates thousands of lines per session, slows performance, and floods the Output Window making real errors harder to spot.

Common Compile Errors

ErrorCauseFix
CS0246: Type or namespace not found Missing using directive or typo in class name Add missing using statement. Verify NinjaTrader namespace spelling exactly.
CS1002: ; expected Missing semicolon at end of statement Add semicolons. Check line indicated in error message.
CS0103: Name does not exist Variable referenced before declaration or wrong scope Declare the variable at class level (private field) if used across methods.
CS0029: Cannot implicitly convert type Assigning a double to an int (or vice versa) without explicit cast Add explicit cast: (int)doubleValue or (double)intValue
CS1061: Does not contain definition for method Calling a method that doesn't exist on the object type Check NinjaTrader 8 API documentation for correct method name and object type.
Strategy already exists / locked DLL Previous version still loaded in NinjaTrader Disable all strategies using that DLL, then recompile.

Runtime Exceptions

Runtime exceptions occur after successful compilation — the strategy loads but throws an error during execution. These appear as red text in the Output Window and typically disable the strategy.

ExceptionCauseFix
IndexOutOfRangeException Referencing Close[N] or indicator[N] before N bars have loaded Add if (CurrentBar < N) return; guard at top of OnBarUpdate
NullReferenceException Indicator or object referenced before initialization (State.DataLoaded not reached) Ensure indicators are initialized in State.DataLoaded, not referenced in Configure
InvalidOperationException on order submit Attempting to submit order outside of valid order submission context Only submit orders inside OnBarUpdate or valid order handlers (OnOrderUpdate, OnExecutionUpdate)
DivideByZeroException Division by a variable that evaluates to zero (e.g. ATR at start of session) Add null/zero guard: if (atr[0] == 0) return; before division operations 
// Defensive coding pattern — wrap risky logic in try-catch during development
try
{
    double ratio = someValue / atr[0];  // risky if ATR = 0
    Print("Ratio: " + ratio);
}
catch (Exception ex)
{
    Print("ERROR in OnBarUpdate Bar " + CurrentBar + ": " + ex.Message);
}

Order State Debugging & Fill Confirmation

When orders aren't filling as expected, use OnOrderUpdate() and OnExecutionUpdate() to trace the full order lifecycle.

// Trace every order state change
protected override void OnOrderUpdate(Order order, double limitPrice,
    double stopPrice, int quantity, int filled,
    double avgFillPrice, OrderState orderState, ...)
{
    Print(Time[0] + " ORDER UPDATE: " + order.Name +
          " State: "  + orderState.ToString() +
          " Filled: " + filled +
          " Avg: "    + avgFillPrice);
}

// Confirm every execution
protected override void OnExecutionUpdate(Execution execution,
    string executionId, double price, int quantity,
    MarketPosition marketPosition, string orderId, DateTime time)
{
    Print("EXECUTION: " + execution.Name +
          " @ "    + price.ToString("F2") +
          " Qty: " + quantity +
          " Side: " + marketPosition.ToString());
}

Order Not Closing Position — Checklist

  • Confirm entry signal name matches exit signal name exactly (case-sensitive)
  • Verify Position.MarketPosition is Long or Short (not Flat) before exit call
  • Check that exit order isn't being blocked by existing stop/target orders
  • Confirm strategy is not in a state where BarsInProgress != 0 is returning early

Folder Structure & DLL Placement

NinjaTrader 8 has strict folder placement requirements. Files in the wrong location will not load regardless of how they are compiled.

Correct File Locations

# NinjaTrader 8 custom file structure
Documents\NinjaTrader 8\
├── bin\
│   └── Custom\
│       ├── Strategies\          ← .cs strategy source files go here
│       ├── Indicators\          ← .cs indicator source files go here
│       └── AddOns\              ← add-on source files
├── bin\
│   └── Custom\
│       └── [Compiled outputs]   ← auto-generated by NinjaTrader compiler
└── log\                         ← NinjaTrader log files for troubleshooting

DLL (Pre-Compiled) Installation

  • DLL files are placed in Documents\NinjaTrader 8\bin\Custom\ (root, not a subfolder)
  • After placing the DLL: Tools → Reload NinjaScript Assemblies
  • Strategy should appear in the strategy list without manual compilation
  • If DLL is locked (in use): disable all strategies, close all charts, then reload
⚠️

Never place .cs files directly in the NinjaTrader 8 root or bin folder. They must be in the correct subfolder (Strategies, Indicators, etc.) or NinjaTrader will not recognize them. This is the most common new-user deployment error.

✅

When in doubt about file location, use Tools → Open NinjaScript Folder in NinjaTrader — this opens the exact directory where custom files should be placed.
