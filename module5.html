Strategy Installation in NinjaTrader

NinjaTrader strategies are distributed as .cs source files or compiled .dll assemblies. Source files require compilation inside NinjaTrader; DLLs are pre-compiled and load directly.

Installing from Source (.cs File)

01

Open NinjaScript Editor

Tools → NinjaScript Editor (or F12). This opens the Visual Studio-style code editor.

02

Navigate to Strategies Folder

In the file tree: NinjaTrader 8 → Custom → Strategies. This is where .cs files must be placed.

03

Place the .cs File

Copy or drag the strategy .cs file into: Documents\NinjaTrader 8\bin\Custom\Strategies\

04

Compile

In NinjaScript Editor: click the Compile button (F5) or right-click → Compile. Watch the Output window for errors.

05

Verify in Strategy List

Open a chart → Strategies tab → the compiled strategy should appear in the list.

⚠️

Folder placement is the #1 deployment error. If a .cs file is placed outside the correct directory path, NinjaTrader will not find it even after compilation. Always use Documents\NinjaTrader 8\bin\Custom\Strategies\.

Instrument & Data Feed Configuration

Incorrect instrument configuration is one of the most common causes of strategies that compile correctly but behave unexpectedly in live trading.

Critical Settings to Verify

  • Instrument Name: Must exactly match what your data provider supplies. MNQ vs MNQ 03-25 vs MNQ1! — these are different instruments.
  • Data Provider: Ensure your chart's data feed matches your brokerage connection. Rithmic and Continuum feeds have different instrument naming.
  • Session Template: Set to CME US Index Futures ETH or RTH depending on your intended trading hours.
  • Tick Size: Verify TickSize in the instrument definition matches what your strategy assumes (0.25 for MES/MNQ).
  • Contract Month / Rollover: Confirm you are on the front-month contract. Stale back-month data causes fill and pricing issues.
// In OnStateChange SetDefaults — explicit instrument validation
// Add this to catch mismatched instrument configuration at startup
if (State == State.DataLoaded)
{
    if (TickSize != 0.25)
    {
        throw new InvalidOperationException(
            "TickSize mismatch. Expected 0.25. Check instrument settings."
        );
    }
}

Simulated vs Live Trading Mode

Always run new strategies in Simulated mode for a minimum of 1–2 weeks before switching to Live. Sim mode uses real market data with paper money — it's the closest approximation to live trading without financial risk.

Enabling a Strategy on a Chart

01

Right-click chart → Strategies

Opens the strategy management panel for that chart.

02

Click Add

Select your compiled strategy from the dropdown list.

03

Configure Parameters

Set all input parameters — lot size, stop loss, profit target, session filter — in the properties panel.

04

Select Account

Choose Sim101 (simulated account) for paper trading. Choose your live brokerage account for live trading.

05

Enable the Strategy

Toggle the strategy to Enabled. Monitor the first few trades for correct behavior.

💡

Sim mode fills are not identical to live fills — they assume perfect execution at signal price. Slippage in live trading will slightly reduce performance vs sim results. Factor in 1–2 ticks of slippage when evaluating sim period performance.

Sim → Live Transition Checklist

  • Strategy ran profitably in sim for minimum 2 weeks across multiple market conditions
  • No unexpected position states observed (stuck positions, double entries)
  • Session filters confirmed working — no overnight trades if not intended
  • Daily loss limit triggers confirmed working
  • Start live with minimum contract size (1 contract) regardless of account size

Common Deployment Errors & Fixes

  • Strategy not appearing in list after compilation: File placed in wrong directory. Move .cs file to Documents\NinjaTrader 8\bin\Custom\Strategies\ and recompile.
  • Compile error — namespace not found: Missing using directives at top of file. Ensure all required NinjaTrader namespaces are included.
  • Strategy loads but no trades execute: Check session filter — strategy may be outside its allowed trading window. Check the Output window for any exception messages.
  • Orders placed but not filled in sim: Limit orders in sim mode require price to trade through (not just touch) the limit level. Switch to market orders for testing order flow.
  • Instrument mismatch error at startup: Strategy was designed for MNQ but is running on MES chart (or vice versa). Reattach to the correct instrument chart.
  • Data feed disconnect mid-session: Strategy may continue running with stale data. Always configure NinjaTrader alerts for connection status changes.
  • Position stuck open at end of session: Exit logic not triggering. Check if ExitLong/ExitShort signal names match entry signal names exactly.
✅

The Output Window (View → Output Window) is your first diagnostic tool. Any strategy exception, warning, or Print() statement appears here. Check it immediately after enabling a strategy.

Real-Time Monitoring Best Practices

Automated doesn't mean hands-off. Effective monitoring catches issues before they compound into significant losses.

NinjaTrader Alert Configuration

  • Set email alerts for strategy enable/disable state changes
  • Configure alerts for connection status changes on your data feed
  • Use Alert() calls within NinjaScript to trigger alerts on unusual conditions

End-of-Day Review Routine

  • Review all trades in the Trade Performance tab
  • Compare actual fills vs expected signal prices — monitor slippage trends
  • Log daily P&L and running drawdown in a spreadsheet
  • Note any market conditions that seemed to cause unusual behavior
  • Check for any error messages in the NinjaTrader log file
⚠️

Never leave automated strategies running unmonitored on a live account during high-impact economic data releases (FOMC, CPI, NFP). Slippage during news events can be 10–50× normal — most scalping strategies should be paused or have reduced size during these windows.
