// 01.01

Strategy Lifecycle: OnStateChange

Every NinjaTrader 8 strategy runs through a defined lifecycle controlled by the OnStateChange() method. Understanding each state is essential before writing any trading logic.

State.SetDefaults
Initialize parameters and display properties. Runs once when strategy is loaded.
State.Configure
Add data series, set calculation mode, configure indicator dependencies.
State.DataLoaded
Data is ready. Initialize indicators and variables that depend on loaded bars.
State.Terminated
Strategy is stopping. Clean up resources, close connections, reset state.
protected override void OnStateChange()
{
    if (State == State.SetDefaults)
    {
        Description = "My NinjaTrader Strategy";
        Name        = "MyStrategy";
        // Define input parameters
        FastPeriod  = 9;
        SlowPeriod  = 21;
        StopLossTicks = 16;
    }
    else if (State == State.Configure)
    {
        // Set calculation mode
        Calculate = Calculate.OnBarClose;
    }
    else if (State == State.DataLoaded)
    {
        // Initialize indicators here
        fastEma = EMA(FastPeriod);
        slowEma = EMA(SlowPeriod);
        atr     = ATR(14);
    }
    else if (State == State.Terminated)
    {
        // Cleanup if needed
    }
}
✅

Best Practice: Always initialize indicators in State.DataLoaded, not State.Configure. Indicators require historical data to exist before initialization.

// 01.02

OnBarUpdate() — Your Strategy Engine

OnBarUpdate() is called on every bar or tick depending on your Calculate mode. All trading logic lives here. The first rule: always guard against insufficient bars.

protected override void OnBarUpdate()
{
    // Guard: wait for enough bars to calculate indicators
    if (CurrentBar < 20) return;

    // Only trade in the primary data series
    if (BarsInProgress != 0) return;

    // Example: EMA crossover signal
    bool bullCross = CrossAbove(fastEma, slowEma, 1);
    bool bearCross = CrossBelow(fastEma, slowEma, 1);

    if (Position.MarketPosition == MarketPosition.Flat)
    {
        if (bullCross) EnterLong("EMA_Long");
        if (bearCross) EnterShort("EMA_Short");
    }
}
⚠️

Common Mistake: Forgetting the CurrentBar < N guard causes index-out-of-range errors when referencing historical bars like Close[1] before enough data exists.

// 01.03

Order Entry Methods

NinjaTrader 8 provides built-in methods for market, limit, and stop orders. For most automated strategies, market orders provide the fastest execution.

Market Orders

// Enter long at market — 1 contract
EnterLong(1, "SignalName");

// Enter short at market — 1 contract
EnterShort(1, "SignalName");

// Exit entire long position at market
ExitLong("ExitSignal", "EntrySignal");
ExitShort("ExitSignal", "EntrySignal");

Limit Orders

// Enter long with a limit price
double limitPrice = Close[0] - (TickSize * 4);
EnterLongLimit(1, limitPrice, "LimitLong");

// Enter short with a limit price
double shortLimit = Close[0] + (TickSize * 4);
EnterShortLimit(1, shortLimit, "LimitShort");
💡

Use signal names consistently between entry and exit calls. Mismatched names are one of the most common causes of positions not closing correctly.

Position Management

Use the Position object to check current market state before entering or exiting trades. Always check position state to avoid duplicate entries.

// Check if flat (no open position)
if (Position.MarketPosition == MarketPosition.Flat)
{
    // Safe to enter
}

// Check if long
if (Position.MarketPosition == MarketPosition.Long)
{
    // Manage or exit long
    if (exitCondition) ExitLong();
}

// Check quantity and average entry price
int    qty        = Position.Quantity;
double avgPrice   = Position.AveragePrice;
double unrealized = Position.GetUnrealizedProfitLoss(PerformanceUnit.Currency, Close[0]);

Indicator Integration

NinjaTrader includes dozens of built-in indicators. Declare them as private fields, initialize in State.DataLoaded, and reference their values in OnBarUpdate().

// Private fields — declare at class level
private EMA  fastEma;
private EMA  slowEma;
private ATR  atr;
private RSI  rsi;
private MACD macd;

// Initialize in State.DataLoaded
fastEma = EMA(9);
slowEma = EMA(21);
atr     = ATR(14);
rsi     = RSI(14, 3);
macd    = MACD(12, 26, 9);

// Reference in OnBarUpdate
double fastVal  = fastEma[0];   // current bar
double fastPrev = fastEma[1];   // previous bar
double rsiVal   = rsi[0];
double atrVal   = atr[0];
// 01.06

ATR-Based Dynamic Stop-Loss & Profit Targets

Hard-coded tick stops are fragile. ATR-based stops adapt to current market volatility — wider in volatile conditions, tighter in quiet markets.

// ATR multiplier parameters (set in SetDefaults)
public double StopAtrMultiplier  = 1.5;
public double TargetAtrMultiplier = 2.5;

// In OnBarUpdate — calculate dynamic levels
double atrVal     = atr[0];
double stopDist   = atrVal * StopAtrMultiplier;
double targetDist = atrVal * TargetAtrMultiplier;

// Convert to ticks for SetStopLoss / SetProfitTarget
int stopTicks   = (int)(stopDist   / TickSize);
int targetTicks = (int)(targetDist / TickSize);

SetStopLoss  ("EntrySignal", CalculationMode.Ticks, stopTicks,   false);
SetProfitTarget("EntrySignal", CalculationMode.Ticks, targetTicks);
✅

Risk/Reward Tip: A 1.5× ATR stop with a 2.5× ATR target gives a 1:1.67 R:R ratio minimum. Most profitable scalping strategies require at least 1:1.5 to remain viable after commissions.
